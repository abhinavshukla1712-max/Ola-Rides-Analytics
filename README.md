## 📌 Project Overview

This is an end-to-end data analytics project on **Ola ride-booking data**. The goal was to analyze ride patterns, cancellations, revenue, driver ratings, and customer behavior using **MySQL** for data querying and **Power BI** for interactive dashboards.

> 💡 This is one of my first data analytics projects as I transition from an accounting background into the data world!

---

## 🎯 Objectives

- Analyze successful vs. cancelled bookings
- Understand revenue trends across vehicle types
- Identify top customers by ride frequency
- Evaluate driver and customer ratings
- Explore payment method preferences

## 📊 Power BI Dashboard Pages

The dashboard contains **5 pages**:

1. **Overall** — High-level summary of all ride bookings
2. **Vehicle Type** — Breakdown of rides and distance by vehicle category
3. **Revenue** — Booking value analysis and payment methods
4. **Cancellation** — Customer vs. driver cancellation reasons
5. **Ratings** — Driver and customer rating analysis

---

## 🗃️ SQL Queries & Views

10 business questions were answered using MySQL views:

| # | Question |
|---|----------|
| 1 | Retrieve all successful bookings |
| 2 | Average ride distance per vehicle type |
| 3 | Total cancelled rides by customers |
| 4 | Top 5 customers by number of rides |
| 5 | Rides cancelled by drivers (personal/car issues) |
| 6 | Max & min driver ratings for Prime Sedan |
| 7 | All rides paid via UPI |
| 8 | Average customer rating per vehicle type |
| 9 | Total booking value of successful rides |
| 10 | Incomplete rides with reasons |
